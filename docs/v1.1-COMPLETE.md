## 🎯 What's New in v1.1

### Watch Mode for Auto-Export ⭐ NEW!

Automatically export prompts when the database changes. Perfect for:
- Keeping `SKILL.md` always up-to-date
- Auto-updating `.cursorrules` in your projects
- CI/CD integration
- Documentation automation

**Usage:**
```bash
# Basic usage
promptvault watch --format skill.md --output SKILL.md

# With custom interval
promptvault watch --format cursorrules --output .cursorrules --interval 2s

# Filter by stack
promptvault watch --format skill.md --output SKILL.md --stack frontend/react

# Verbose mode
promptvault watch -f skill.md -o SKILL.md -v
```

**Example Output:**
```
ℹ  Performing initial export...
✓ Watching for changes... (interval: 5s)
ℹ  Export format: skill.md → SKILL.md
ℹ  Press Ctrl+C to stop

ℹ  Detected change (#1), exporting...
✓ Exported SKILL.md

ℹ  Detected change (#2), exporting...
✓ Exported SKILL.md
```

**Features:**
- ✅ Initial export on startup
- ✅ Configurable check interval (default: 5s)
- ✅ Graceful shutdown with Ctrl+C
- ✅ Session export counter
- ✅ Stack filtering support
- ✅ Verbose/debug logging
- ✅ Error handling with retries

## 🎓 Usage Examples

### 1. Complete Workflow with Watch Mode

```bash
# Terminal 1: Start watching
promptvault watch --format skill.md --output SKILL.md

# Terminal 2: Add prompts
promptvault add "React Hook" --content "useEffect example..."
# → Watch mode auto-exports to SKILL.md

# Terminal 2: Edit a prompt
promptvault # Open TUI and edit
# → Watch mode detects change and re-exports
```

### 2. CI/CD Integration

```bash
# In CI pipeline
promptvault watch --format cursorrules --output .cursorrules &
WATCH_PID=$!

# Run tests, make changes
npm test

# Add new prompts based on learnings
promptvault add "Test pattern" --content "..."

# Stop watch mode
kill $WATCH_PID
```

### 3. Multi-Stack Watching

```bash
# Watch React prompts only
promptvault watch --format skill.md --output REACT.md --stack frontend/react

# Watch all Python prompts
promptvault watch --format agents.md --output PYTHON.md --stack backend/python
```
